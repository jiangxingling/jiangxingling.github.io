App测试之压力测试

发表于 2017-04-26

概念

压力测试就是让手机在各种各样的情况下进行测试，以发现不容易被发现的缺陷
在压力测试下可能出现的异常情况：

CRASH
ANR

使用的工具

Monkey
Monkey Script       –用来重复执行操作
Monkey Runner

测试步骤

  1. 确保手机连上电脑
    adb devices //查看设备

  2. 发送压力指令
    adb shell monkey 1000 //表示对设备随机发送1000个指令
    adb shell monkey -p 1000 //表示对指定的包名发送1000个指令

  3. 获取包名的方式
    adb shell pm list package //查看所有包名，不需要root
    adb shell dumpsys window w | findstr \/ | findstr name= //直接查看应用的包名

  4. 如果最后出现Events injected:1000，表示所有指令都已经成功发送，没有异常

  5. 查看当前运行的Activity名称
    进入adb shell
    运行logcat | grep ActivityManager

高级参数

throttle

指定事件之间的时间间隔
adb shell monkey -p < package > –throttle < milliseconds > 100

seed

指定操作的序列值，便于复现操作
adb shell monkey -p -s 50

触摸事件占比

设置触摸事件的百分比
adb shell monkey -v -p < package > –pct-touch < percent > 100

-v表示打印事件详情

如：adb shell monkey -v -p –pct-touch 100 100

动作事件占比

设置动作事件所占百分比
adb shell monkey -v -p < package > –pct-motion 80 –pct-touch 20 100

轨迹球占比

adb shell monkey -p < package > –pct-trackball < percent > 1000

基本导航事件占比

adb shell monkey –pct-nav < percent > 1000

主要导航事件（包括中间键、返回键、菜单按键等）占比

adb shell monkey –pct-majornav < percent > 1000

系统导航事件（包括HOME、BACK、拨号、音量键）占比

adb shell monkey –pct-syskeys < percent > 1000

启动Activity占比

adb shell monkey –pct-appswitch < percent > 1000

不常用事件占比

adb shell monkey –pct-anyevent < percent > 1000

崩溃事件

在发生崩溃时忽略并继续执行指令
adb shell monkey –ginore-crashes 1000

超时事件

出现ANR时继续执行
adb shell monkey –ignore-timeouts 1000

ANR信息

log存放在data/anr/trace.txt文件中

Monkey Script

adb shell monkey -f < scriptfile > < event-count >
scriptfile是脚本文件，event-count表示执行次数

    轨迹球事件。action为0代表按下，为1代表弹起，x/y代表坐标

    输入字符串

  • LaunchActivity(package, Activity)
    启动应用

  • UserWait(int milliseconds)
    等待

  • DispatchPress(int keycode)
    按下键值，回车键值为66

RandomAccessFile

概念

RandomAccessFile 提供对文件内容的访问，既可以读文件，也可以写文件
支持随机访问文件，可以访问文件的任意位置

(1)Java文件模型
在硬盘上的文件是byte byte byte存储的，是数据的集合
(2)打开文件
有两种模式，”rw”读写，”r”只读
RandomAccessFile raf = new RandomAccessFile(file,”rw”)
文件指针，打开文件时指针在开头 pointer = 0
(3)写方法
raf.write(int) —> 只写一个字节，同时指针指向下一个位置
(4)读方法
int b = raf.read() —> 读一个字节
(5)文件读写完后一定要关闭
raf.close()

创建对象

RandomAccessFile raf = new RandomAccessFile(file3,"rw");

写入数据

//指针的位置
System.out.println("刚开始时指针的位置：" + raf.getFilePointer());

//写入一个字符
raf.write('A');
System.out.println("写入A后指针的位置：" + raf.getFilePointer());

//写入字符串
String s = "Hello World!";
byte[] data = s.getBytes();
raf.write(data);
System.out.println("写入'Hello World!'后指针的位置：" + raf.getFilePointer());

//写入整数（因为一次只能写入一个字节，所以要从高八位开始依次写到低八位）
int num = 100;
raf.write(num >>> 24);
raf.write(num >>> 16);
raf.write(num >>> 8);
raf.write(num);
//或者
raf.writeInt(num);
System.out.println("写入整数后指针的位置：" + raf.getFilePointer());

读取数据

//读文件（必须先把指针移到头部）
raf.seek(0);

//一次性读取 
byte[] buf = new byte[(int)raf.length()];
raf.read(buf);
System.out.println("一次性读取并输出：" + Arrays.toString(buf));

//或者以字符串形式输出（无法读取整数）
String s1 = new String(buf);
System.out.println("转换成字符串输出：" + s1);

//读取整数
int n1 = raf.readInt();
//或者：
int n2 = (int) (buf[0] << 24|buf[1] << 16 | buf[2] << 8 | buf[3]);

关闭文件

raf.close();

IO流

概念

IO流分为输入流，输出流，也可以分为字节流，字符流

字节流

EOF和-1表示到达文件结尾

InputStream

抽象了应用程序读取数据的方式
基本方法：

int b = in.read() –读取一个字节无符号填充到int低八位
in.read(byte[] buf) –读取数据填充到字节数组buf中
in.read(byte[] buf, int start, int size) –从start开始读取size个字节到buf中

FileInputStream

是InputStream的子类，具体实现在文件上读取数据

  • 一个字节一个字节的读取输出 

    //把文件作为字节流进行读操作
FileInputStream in = new FileInputStream(fileName);
int b;
int i = 1;
System.out.println("每次输出一个字节：");
while((b = in.read()) != -1){
    System.out.print(b + " ");
    //每读取10个字节换行
    if(i++ % 10 == 0){
        System.out.println();
    }
}
System.out.println();
in.close();

  • 批量读取输出 

    FileInputStream in = new FileInputStream(fileName);
byte[] buf = new byte[1024];
//从in中批量读取字节放入到buf中，返回读到的字节的个数
int bytes = in.read(buf, 0, buf.length);
int j = 1;
System.out.println("批量读取：");
for(int i = 0; i < bytes; i++){
    System.out.print((buf[i] & 0xff) + " ");
    if(j++ % 10 == 0){
        System.out.println();
    }
}
System.out.println();
in.close();
DataInputStream

对流功能的扩展，可以更加方便的读取int、long、字符等类型数据

  • 读取数据 

    public static void readData(File file) throws IOException{
    DataInputStream dis = new DataInputStream(
            new FileInputStream(file));

    int n = dis.readInt();
    String str = dis.readUTF();
    dis.close();

    System.out.println(n + str);
}
BufferedInputStream

提供了带缓冲区的操作，提高了IO的性能

  • 拷贝文件

    public static void copyFileByBuffer(File srcFile, File destFile) throws IOException{
    if(!srcFile.exists()){
        System.out.println(srcFile + "文件不存在");
    }else if(!srcFile.isFile()){
        System.out.println(srcFile + "不是文件");
    }else{
        BufferedInputStream in = new BufferedInputStream(
                new FileInputStream(srcFile));
        BufferedOutputStream out = new BufferedOutputStream(
                new FileOutputStream(destFile));

        int c;
        while((c=in.read())!=-1){
            out.write(c);
            out.flush();  //刷新缓冲区
        }
        in.close();
        out.close();
    }
}
OutputStream

抽象了应用程序写出数据的方式
基本方法：

out.write(int b) –> 写出一个byte（b的低八位）到流
out.write(byte[] buf) –> 将buf字节数组都写入到流
out.write(byte[] buf, int start, int size) –> 从buf中的start开始写size个字节到流

是OutputStream的子类，实现向文件中写出byte数据的方法

    public static void copyFile(File srcFile, File destFile) throws IOException{
    if(!srcFile.exists()){
        System.out.println(srcFile + "文件不存在");
    }else if(!srcFile.isFile()){
        System.out.println(srcFile + "不是文件");
    }else{

Android之屏幕适配

发表于 2017-04-19

基本概念

屏幕尺寸、屏幕分辨率、屏幕像素密度

  • 屏幕尺寸

    指屏幕的对角线长度，单位是英寸，1英寸约等于2.54cm

  • 屏幕分辨率

    指在横纵向上的像素点数，单位是px，1px = 1个像素点
    一般是纵向像素 横向像素，如1920 1080

  • 屏幕像素密度

    指每英寸上的像素点数，单位是dpi (dot per inch)
    屏幕像素密度与屏幕尺寸和屏幕分辨率有关
    计算方式（以4.9英寸，19201080的手机为例）：dpi = √(19201920+1080*1080) / 4.9

dp、dip、dpi、sp、px

  • dp、dip

    Density Independent Pixels的缩写，即密度无关像素。
    以160dpi为基准，1dip = 1px，在320dpi上，1dip = 2px
    使用dp/dip可以保证在不同设备上显示相同的效果

  • dpi

    即屏幕像素密度

  • sp

    Scale-Independent Pixels
    常用来设置文字大小

  • px

    构成图像的最小单位。如获取到的屏幕宽高是以px为单位的

mdpi、hdpi、xdpi、xxdpi

  • 对应关系

    mdpi – 120dpi ~ 160dpi
    hdpi – 160dpi ~ 240dpi
    xdpi – 240dpi ~ 320dpi
    xxdpi – 320dpi ~ 480dpi
    xxxdpi – 480dpi ~ 640dpi

Android之内存优化

发表于 2017-04-17

为什么要进行内存优化

在实际开发中，不再用到的对象因为被错误引用导致无法回收，从而造成内存泄漏，甚至内存溢出OOM(OutOfMemory)，程序崩溃。

查看手机内存限制

  1. 获取系统服务和内存限制

  2. 将读取的数据放入StringBuilder中并打印出来

Android 系统内存管理方式

系统内存分配与回收方式

  • 每个App都相当于一个进程对应一个虚拟机

  • 使用GC进行回收

    GC是Java 垃圾处理器的简称，它只在Heap剩余空间不够时才发起垃圾回收动作。当GC被触发时，所有的线程都会被暂停。

App内存限制机制

  • 每个App分配的最大内存限制，根据设备不同而不同
  • 占用内存最厉害的是图片

切换应用时后台App清理机制

  • App 切换时依据LRU Cache
    LRU算法：最近使用的排在最前面，最少可能的被清理掉
  • onTrimMemory()回调方法
    系统内存发生变化时，会调用该方法

监控内存的几种方法

  • 代码监控系统分配给App的内存以及剩余内存

  • 通过Android Monitor监控

  • Tools – Android – Android Device Monitor
    选择要查看的进程，点击Update Heap图标，再点击Cause GC

APP内存优化方法

数据结构优化

  • 频繁字符串拼接用StringBuilder

    通过+的方式进行字符串拼接，会产生中间字符串块，这些都是没有用的

  • 使用ArrayMap、SparseArray替换HashMap
  • 内存抖动
  • 再小的Class也要耗费0.5KB
  • HashMap一个entry需要额外占用32B

对象复用

  • 复用系统自带的资源
  • ListView/GridView的ConvertView复用
  • 避免在onDraw()方法里面执行对象的创建

避免内存泄露

内存泄露：由于代码瑕疵，导致这块内存，虽然已经停止不用了，但依然被其它东西引用着，使得GC没法对它回收。
内存泄露会导致剩余可用的Heap越来越少，频繁触发GC

  • 要注意Activity泄漏
  • 用Application Context而不是Activity Context
  • 注意Cursor对象是否及时关闭

OOM问题优化

强引用与软引用

强引用在其生命周期中不会被GC回收，软引用可以被回收。

优化OOM问题的方法

  • 注意临时Bitmap对象的及时回收
  • 避免Bitmao的浪费
  • Try catch某些大内存分配的操作
  • 加载Bitmap: 缩放比例、解码格式、局部加载
