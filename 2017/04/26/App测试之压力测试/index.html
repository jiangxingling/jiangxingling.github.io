App测试之压力测试

发表于 2017-04-26

概念

压力测试就是让手机在各种各样的情况下进行测试，以发现不容易被发现的缺陷
在压力测试下可能出现的异常情况：

CRASH
ANR

使用的工具

Monkey
Monkey Script       –用来重复执行操作
Monkey Runner

测试步骤

  1. 确保手机连上电脑
    adb devices //查看设备

  2. 发送压力指令
    adb shell monkey 1000 //表示对设备随机发送1000个指令
    adb shell monkey -p 1000 //表示对指定的包名发送1000个指令

  3. 获取包名的方式
    adb shell pm list package //查看所有包名，不需要root
    adb shell dumpsys window w | findstr \/ | findstr name= //直接查看应用的包名

  4. 如果最后出现Events injected:1000，表示所有指令都已经成功发送，没有异常

  5. 查看当前运行的Activity名称
    进入adb shell
    运行logcat | grep ActivityManager

高级参数

throttle

指定事件之间的时间间隔
adb shell monkey -p < package > –throttle < milliseconds > 100

seed

指定操作的序列值，便于复现操作
adb shell monkey -p -s 50

触摸事件占比

设置触摸事件的百分比
adb shell monkey -v -p < package > –pct-touch < percent > 100

-v表示打印事件详情

如：adb shell monkey -v -p –pct-touch 100 100

动作事件占比

设置动作事件所占百分比
adb shell monkey -v -p < package > –pct-motion 80 –pct-touch 20 100

轨迹球占比

adb shell monkey -p < package > –pct-trackball < percent > 1000

基本导航事件占比

adb shell monkey –pct-nav < percent > 1000

主要导航事件（包括中间键、返回键、菜单按键等）占比

adb shell monkey –pct-majornav < percent > 1000

系统导航事件（包括HOME、BACK、拨号、音量键）占比

adb shell monkey –pct-syskeys < percent > 1000

启动Activity占比

adb shell monkey –pct-appswitch < percent > 1000

不常用事件占比

adb shell monkey –pct-anyevent < percent > 1000

崩溃事件

在发生崩溃时忽略并继续执行指令
adb shell monkey –ginore-crashes 1000

超时事件

出现ANR时继续执行
adb shell monkey –ignore-timeouts 1000

ANR信息

log存放在data/anr/trace.txt文件中

Monkey Script

adb shell monkey -f < scriptfile > < event-count >
scriptfile是脚本文件，event-count表示执行次数

常用方法

  • DispatchTrackball(int action, float x, float y)
    轨迹球事件。action为0代表按下，为1代表弹起，x/y代表坐标

  • DispatchPointer(int action, float x, float y)
    点击事件（只需要关注第三至五个参数，其余使用默认，坐标值用uiautomatorviewer获取）

  • DispatchString(String text)
    输入字符串

  • LaunchActivity(package, Activity)
    启动应用

  • UserWait(int milliseconds)
    等待

  • DispatchPress(int keycode)
    按下键值，回车键值为66

Monkey Runner API

  • alert –警告框
    void alert(String message, String title, String okTitle)
    from com.android.monkeyrunner import MonkeyRunner MonkeyRunner.alert(“this is alert”,”Hi”,”OK”)

  • waitForConnection(float timeout, String deviceId)
    等待设备连接，有多个deviceid时需要指明具体哪个设备。timeout为超时时间，以秒为单位

MonkeyDevice API

  • drag(tuple start, tuple end, float duration, integer steps)
    在操作页面上进行拖动
    duration为手势持续时间
    steps为插值点的步数，默认为10

  • press(String keyCode, dictionary type)
    keyCode为键值
    type为类型，可以是DOWN/UP/DOWN_AND_UP

  • startActivity(package+”/“+Activity)
    启动APP

  • touch(integer x, integer y, integer type)
    type为DOWN/UP/DOWN_AND_UP

  • type(String message)
    输入字符串

  • MonkeyImage takeSnapshot()
    截屏

MonkeyImage API

  • boolean sameAs(MonkeyImage other, float percent)
    对比图像

  • writeToFile(String path, String format)
    保存文件图像
    path指定保存路径，format指定保存格式，如png、jpg等
